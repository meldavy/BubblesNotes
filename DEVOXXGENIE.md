## Project Guidelines

### Build Commands

- **Build:** `./gradlew.bat build`
- **Test:** `./gradlew.bat test`
- **Single Test:** `./gradlew.bat test --tests ClassName.methodName`
- **Clean:** `./gradlew.bat clean`
- **Run:** `./gradlew.bat run`

### Code Style

- **Formatting:** Use IDE or checkstyle for formatting
- **Naming:**
  - Use camelCase for variables, methods, and fields
  - Use PascalCase for classes and interfaces
  - Use SCREAMING_SNAKE_CASE for constants
- **Documentation:** Use JavaDoc for documentation
- **Imports:** Organize imports and avoid wildcard imports
- **Exception Handling:** Prefer specific exceptions and document throws

### When encountering back-to-back issues compiling

- Reference other code in the workspace for examples
- Reference official documentation from the web for help
